## Purpose

Projman is a demand tracking and delivery governance tool that helps small IT teams manage work from initial idea through deployment and completion. It is designed for teams that need to balance incoming requests against limited resources and match project skill requirements with available staff. The application solves the problem of lost or mismanaged work requests, unclear ownership, and staff overload by providing lifecycle tracking, skills-based staffing, workload visualization, and audit history.

## Tech Stack

Laravel 12, PHP 8.4, livewire, flux, flux-pro

## Features (from tests)

Maintains consistent role and user-role state when roles are updated, deactivated, or deleted  
Submits project scheduling details to DCGG and records audit/history while emitting scheduling events  
Schedules project deployment and records history while emitting scheduling events  
Generates a staff heatmap of activity across upcoming working days for active projects  
Staff can view and edit their skills  
Admins can create, edit, and delete skills and assign skills with levels to users, with user counts per skill  
Admins can manage users, search/sort them, and change their roles via a modal  
Saves effort scale and submits project scoping, emailing Work Package Assessors and recording history  
Displays a project staffing heatmap that highlights assigned staff (including technical lead, change champion, and CoSE IT) and integrates with the project model view  
Advances a project to the next lifecycle stage (including via Livewire)  
Admins can view a user overview including details, roles, skills, requests, and IT assignments  
Supports toggling inclusion of completed and cancelled IT assignments in views  
Matches users to required skills, scoring and sorting staff by most applicable competency  
Creates a project and all related lifecycle sub-forms (ideation, feasibility, scoping, scheduling, detailed design, development, testing, deployed)  
Approves or rejects feasibility with readiness checks, notifications, emails, and recorded history  
Sends domain notifications for project creation, feasibility decisions, scoping submission, scheduling milestones, and stage changes to role-based recipients  
Shows user roles, counts, and admin status, and allows interactive role reassignment with badges and tooltips
