## Purpose

Projman is a demand tracking and delivery governance tool that helps small IT teams manage projects from initial idea through deployment and completion. It is aimed at teams that need to match project work with staff skills while balancing limited capacity. The application solves the problem of lost or poorly managed work requests by providing lifecycle tracking, skills-based staffing, workload visualization, and audited project history.

## Tech Stack

Laravel 12, PHP 8.4, livewire, flux, flux-pro

## Features (from tests)

Maintains consistent role assignments and user counts when roles are updated or deactivated  
Submits project scheduling to DCGG and records history and audit trail  
Schedules change board dates for projects and records history  
Can create and manage roles (create, update, delete)  
Tracks user counts per role and updates counts as users are assigned or removed  
Allows disabling project development while still showing build information in scoping  
Staff can view and edit their skills and proficiency levels  
Admins can manage the skills catalogue and assign skills with levels to users  
Displays an admin user overview with user details, roles, skills, requests, and IT assignments  
Submits scoping with an effort scale, notifies Work Package Assessors, and records history  
Generates a heatmap of staff activity for project staffing and busyness  
Manages technical lead, change champion, and change board outcome fields within scheduling  
Can advance a project to the next delivery stage (including via Livewire)  
Matches users to required competencies by skill level and score for staffing decisions  
Approves or rejects feasibility with email notifications, history, and approval status badges  
Can create a project and all related lifecycle forms (ideation through deployed)  
Sends lifecycle notifications for project creation, feasibility decisions, scoping submission, scheduling milestones, and stage changes to configured roles  
Displays and manages user roles, including badges, assignment, and real-time updates in the UI  
Admin users can assign and unassign roles to users, including handling many roles and tooltips
